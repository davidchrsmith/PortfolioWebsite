# Personal Portfolio

My personal portfolio website built with React, Three.js, and Tailwind CSS. It covers my projects, experience, skills, and has a contact form wired up through EmailJS.

## Tech Stack

- **React 19** — UI
- **Vite** — build tool / dev server
- **Three.js + React Three Fiber** — 3D scene in the hero section
- **Tailwind CSS v4** — styling
- **EmailJS** — contact form (no backend needed)

---

## Local Development

Clone the repo and install dependencies:

```bash
npm install
```

Start the dev server:

```bash
npm run dev
```

Open [http://localhost:5173/PortfolioWebsite/](http://localhost:5173/PortfolioWebsite/) in your browser.

> **Note:** The site is served under the `/PortfolioWebsite/` subpath (set via `base` in `vite.config.js`) to match how GitHub Pages serves it. Make sure to include that path when visiting locally.

## Deployment (GitHub Pages)

This site is deployed automatically via **GitHub Actions**. Every push to `main` triggers a workflow that builds the project and pushes the compiled output to the `gh-pages` branch — GitHub Pages then serves from that branch.

### How it works

- `main` branch — source code only (what you edit)
- `gh-pages` branch — built output only (auto-generated by CI, do not edit manually)
- GitHub Pages is configured to serve from the `gh-pages` branch
- The workflow file lives at `.github/workflows/deploy.yml`

### Environment variables (EmailJS)

EmailJS credentials are stored as **GitHub Actions Secrets** (never committed to the repo). The workflow injects them at build time so Vite can bake them into the bundle.

Required secrets (set under **Settings → Secrets and variables → Actions**):
- `VITE_EMAILJS_SERVICE_ID`
- `VITE_EMAILJS_TEMPLATE_ID`
- `VITE_EMAILJS_PUBLIC_KEY`

For local development, create a `.env.local` file in the project root (gitignored) with the same keys:

```
VITE_EMAILJS_SERVICE_ID=your_service_id
VITE_EMAILJS_TEMPLATE_ID=your_template_id
VITE_EMAILJS_PUBLIC_KEY=your_public_key
```

### First-time setup

1. Add the three EmailJS secrets in your repo (**Settings → Secrets and variables → Actions**).
2. Make sure GitHub Pages is configured:
   - Go to **Settings → Pages**
   - Set **Source** to the `gh-pages` branch, folder `/ (root)`

### Updating the live site after making changes

Just push to `main` — the workflow handles the rest:

```bash
git add -A
git commit -m "your message"
git push
```

The live site at `https://davidchrsmith.github.io/PortfolioWebsite/` will update within a minute or two. You can monitor progress under the **Actions** tab in your GitHub repo.
